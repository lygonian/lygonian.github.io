#A01 produktiv

ERP/CRM Incremental Sync Engine

Ein ERP-System mit tausenden Kontakten, Dokumenten und Projekten — und keine einfache Möglichkeit, lokale Analysen damit zu betreiben. Die Lösung: eine inkrementelle Synchronisations-Engine, die per GraphQL nur geänderte Datensätze abruft, lokal in CSV speichert und dabei stabil mit Rate-Limits und Serverfehlern umgeht. Aus stündlichen manuellen Exporten wurde ein vollautomatischer Datenstrom.

Wie es funktioniert
1
Wasserzeichen-basiertes Delta
Der letzte bekannte Zeitstempel wird als Watermark gespeichert. Beim nächsten Lauf werden nur Datensätze abgerufen, die danach geändert wurden.
2
Zweiphasen-Abfrage
Phase 1: Leichte Abfrage (nur IDs + Timestamps) zur schnellen Identifikation geänderter Einträge. Phase 2: Vollständige Abfrage nur für die geänderten IDs in Batches.
3
Loop-until-stable
Synchronisation läuft solange, bis ein Durchlauf keine neuen Änderungen mehr findet — garantiert vollständige Konsistenz auch bei serverseitiger Paginierung.
4
Fehlerresistenz
Automatischer Retry bei HTTP 429/5xx, exponentielles Backoff, sauberes Logging — der Sync erholt sich selbst von transienten Serverfehlern.
import os, time, requests
import pandas as pd
from datetime import datetime, timezone

API_TOKEN = os.environ["CRM_API_TOKEN"]  # niemals hardcoded
API_URL   = os.environ["CRM_API_URL"]

def gql(query, variables=None, retries=3):
    # GraphQL mit automatischem Retry bei 429/5xx
    for attempt in range(retries):
        r = requests.post(
            API_URL,
            json={"query": query, "variables": variables or {}},
            headers={"Authorization": f"Bearer {API_TOKEN}"},
            timeout=30
        )
        if r.status_code == 429:
            time.sleep(2 ** attempt)
            continue
        r.raise_for_status()
        return r.json()["data"]
    raise RuntimeError("Max retries exceeded")

def upsert_by_id(existing_df, new_records):
    # Neue und geänderte Datensätze in lokale CSV einpflegen
    new_df  = pd.DataFrame(new_records)
    merged  = pd.concat([existing_df, new_df])
    return (merged
            .sort_values("modified_at", ascending=False)
            .drop_duplicates(subset=["id"], keep="first")
            .reset_index(drop=True))

def sync_until_stable(entity_fn, csv_path, watermark):
    # Loop: sync → prüfen ob neue Daten → wiederholen bis stabil
    while True:
        new_records, new_watermark = entity_fn(watermark)
        if not new_records:
            break
        df = pd.read_csv(csv_path) if csv_path.exists() else pd.DataFrame()
        df = upsert_by_id(df, new_records)
        df.to_csv(csv_path, index=False)
        watermark = new_watermark

Angebots-Automatisierung — PDF Pipeline

Angebote erstellen: CRM öffnen, PDF herunterladen, Positionen ablesen, Word-Vorlage befüllen, PDF exportieren, ablegen. Für jedes Angebot. Jede Woche. Die Automatisierung übernimmt den gesamten Prozess: CRM-Sync, PDF-Extraktion per pdfplumber, Regex-Parsing der Positionen, Template-Befüllung via pdfrw — vollständig ohne manuelle Eingriffe.

Technische Umsetzung
1
PDF-Textextraktion mit pdfplumber
Strukturierte Textextraktion aus mehrseitigen PDFs. Tabelleninhalt wird seitenweise zusammengeführt und als Rohtext bereitgestellt.
2
Regex-Parsing der Angebotspositionen
Reguläre Ausdrücke extrahieren Positionsnummer, Beschreibung, Menge und Einzelpreis — robust gegen Formatvariationen und mehrzeilige Einträge.
3
Template-Befüllung mit pdfrw
PDF-Formularfelder werden programmatisch mit den extrahierten Werten befüllt. Ergebnis-PDF wird mit Angebotsnummer benannt und archiviert.
4
CSV-Protokoll & Fehlerbehandlung
Jedes verarbeitete Angebot wird mit Zeitstempel, Status und Dateipfad protokolliert. Fehlhafte PDFs werden in einer separaten Queue gesammelt.
import pdfplumber, pdfrw, re
from pathlib import Path
from datetime import datetime

# Positionen aus PDF-Text extrahieren
POSITION_PATTERN = re.compile(
    r'(?P<pos>\d+)\s+'
    r'(?P<desc>[A-Za-z][\w\s,\-\.]+?)\s+'
    r'(?P<qty>\d+(?:[.,]\d+)?)\s+(?:Stk|m²|h)\s+'
    r'(?P<price>\d{1,3}(?:\.\d{3})*(?:,\d{2}))'
)

def extract_positions(pdf_path):
    with pdfplumber.open(pdf_path) as pdf:
        full_text = "\n".join(p.extract_text() or "" for p in pdf.pages)
    return [m.groupdict() for m in POSITION_PATTERN.finditer(full_text)]

def fill_template(template_path, output_path, fields):
    template = pdfrw.PdfReader(template_path)
    for page in template.pages:
        annotations = page.get("/Annots") or []
        for ann in annotations:
            key = str(ann.get("/T", "")).strip("()")
            if key in fields:
                ann.update(pdfrw.PdfDict(V=fields[key], AP=""))
    pdfrw.PdfWriter().write(output_path, template)

Was vorher 15–20 Minuten pro Angebot dauerte, läuft jetzt in unter 10 Sekunden vollautomatisch durch. Formatfehler durch manuelle Übertragung gehören der Vergangenheit an.
